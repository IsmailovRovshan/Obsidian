#### Основные методы

**GET**

- Получает данные с сервера.
- Не изменяет состояние.
- Идемпотентный.
- Пример: `/api/users?age=25`

**POST**

- Создает новый ресурс.
- Неидемпотентный.
- Пример: `/api/users`  
    Тело запроса (JSON):
    ```json
    { "name": "Alex", "email": "alex@example.com" }
    ```

**PUT**

- Полностью обновляет существующий ресурс.
- Идемпотентный.
- Пример: `/api/users/1`

**PATCH**

- Частично обновляет ресурс.
- Может быть идемпотентным (в зависимости от реализации).
- Пример: `/api/users/1`
    
    ```json
    { "email": "new@example.com" }
    ```

**DELETE**

- Удаляет ресурс.
- Обычно идемпотентный.
- Пример: `/api/users/1`

---

#### Дополнительные методы

|Метод|Назначение|
|---|---|
|**HEAD**|Проверить, существует ли ресурс (как GET, но без тела).|
|**OPTIONS**|Узнать, какие методы разрешены для ресурса.|
|**TRACE**, **CONNECT**|Используются редко, в основном для отладки и прокси.|
